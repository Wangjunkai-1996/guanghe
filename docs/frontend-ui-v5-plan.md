# 前端 UI V5 基线说明

## 版本定位

- `V5` 统一作为当前前端 UI 的最新基线。
- 2026-03-15 这轮对话中完成的“恢复上一轮偏好的界面、桌面端暗色控制台化、按钮可读性修复、Google Cloud 风格收口”都并入 `V5`，不再额外拆成新的版本号。
- `V1 / V2 / V3 / V4` 继续保留为历史演进记录，但后续继续做前端页面优化时，应默认以 `V5` 为起点。

## 当前设计目标

- 以“高频运营工具”而不是“展示型后台”为目标。
- 优先保证：
  - 扫读效率
  - 状态判断
  - 下一步动作推进
  - 桌面端批量闭环效率
- 默认不为了“更好看”牺牲任务推进速度。

## V5 当前界面基线

### 1. 全局壳层

- 桌面端默认使用暗色工具台。
- 左侧为高密功能导航，工作区固定为：
  - `批量闭环`
  - `快速验证`
- 顶部保留单条状态栏，只显示：
  - 当前工作区
  - 账号库状态
  - 腾讯文档状态
  - 最近同步

### 2. 批量页

- 批量页按四阶段固定重排：
  1. `交接表准备`
  2. `需求摘要`
  3. `任务发起`
  4. `任务队列`
- 任务区为桌面端主工作面，采用左侧列表 + 右侧详情。
- 任务详情默认先给出“结论”，再进入回填、日志、截图等次级内容。

### 3. 手工页

- 手工页降级为“快速验证工具”，只保留：
  - 账号管理
  - 内容 ID 查询
  - 最近结果复核
- 桌面端结构为：
  - 左侧账号 rail
  - 右侧查询条与结果面板

### 4. 桌面暗色风格

- 风格方向统一为：
  - 石墨灰底
  - 低饱和蓝强调
  - 分层面板
  - 高密可读
  - 克制动效
- 本轮不再追求品牌化 Hero、营销化页头或装饰性说明文案。
- Google Cloud 风格在本项目中的落点重点是：
  - 控制台式导航与状态模块
  - 分层而非发光的深色面板
  - 清晰的主次按钮权重
  - 更接近生产工具的排布节奏

### 5. 本轮补充修复

- 修正桌面暗色按钮文字颜色继承问题，避免按钮背景和文字冲突导致看不清。
- 统一主按钮、次按钮、危险按钮、激活态 chip 的对比度规则。
- 将 `InlineNotice`、阶段提示卡、状态条和侧栏状态卡统一进桌面暗色体系，减少“局部浅色卡片残留”。
- 收窄左侧壳层和状态模块的展示感，强化控制台感。
